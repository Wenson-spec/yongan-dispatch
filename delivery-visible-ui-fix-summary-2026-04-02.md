# 2026-04-02 前端可见改造补齐说明

本轮补做的重点，不是管理驾驶舱首页，而是**找车台、回单管理台、零担统一工作台，以及确认派车相关摘要区域**。因此，你刚才发来的截图停留在“管理驾驶舱”页面时，看不到本轮最主要的改动，这也是“观感不明显”的直接原因。当前首页卡片本身并不是这轮收口的核心展示面。

| 页面/区域 | 本轮已补齐的可见改造 | 现在应看到的效果 |
| --- | --- | --- |
| 找车台 | 将多处直接暴露 `mergedPlanNumber` 的组头与摘要，统一收口为**正式外请分组优先**，仅在必要时保留“参考批次”说明 | 用户优先看到正式外请分组、当前车次或业务说明，而不是旧合并计划号直接充当主标题 |
| 确认派车摘要 | 弹窗与摘要说明中，旧批次号改为**参考批次**语义，避免误解为正式分组 | 摘要区会明确“当前整组派车优先按正式外请分组展示” |
| 回单管理台 | 待收回单、已收回单的分组键与单行提示，优先跟随正式外请分组；若尚未形成正式分组，则回退到同车次口径 | 组头不再把旧批次直接当正式分组；单行会提示“当前归属正式外请分组”或“暂按同车次跟踪/归档” |
| 零担统一工作台 | 待询价区新增执行摘要口径，优先强调**车号、司机、货站、目的站**等内部整理结果，旧批次降级为内部参考信息 | 用户应先看到“车次执行”“货站执行”或“目的站执行”等更直观摘要，而不是只看到旧批次号 |
| 派车台批量派车摘要 | 旧文案“合并计划号”调整为“内部整理参考批次（非正式外请组）” | 减少将旧批次误认成正式外请分组的可能 |

从实现结果看，这轮改造已经把“**正式外请分组**”与“**内部参考批次**”两个概念在前端层面明显拆开。此前用户之所以会觉得改动不明显，核心问题在于：很多变化停留在逻辑和字段选择上，但组头、摘要、单行提示没有同步改得足够直观。现在这部分已经补齐。
