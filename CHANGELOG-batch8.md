# CHANGELOG-batch8

## 概述

本批次围绕零担前后段六模式联动进行调整，核心目标是取消人工模式选择，改为由系统根据实际操作自动推断前段与后段模式，并联动回单责任与外请审批流程。实现重点覆盖后端推断规则、回单自动清理与恢复、零担统一工作台新增入口，以及相关展示文案同步。

## 主要改动

| 模块 | 改动内容 | 说明 |
| --- | --- | --- |
| 后端订单路由 | 补强零担前后段模式标记与推断辅助逻辑 | 基于前段派车、前段外请子链、客户自送到站确认、后段外请子链、客户自提标记等实际操作推断模式 |
| 后端订单编辑接口 | 新增零担客户自提与客户自送到站确认入口 | 通过通用更新接口接入 `ltlCustomerPickup` 与 `ltlCustomerSelfDeliverConfirmed` 两类操作字段，并限制仅零担订单可用 |
| 后端回单联动 | 客户自提时禁止继续承担回单 | 标记客户自提后将主单回单责任切换为 `none`，并清理或停止当前主单回单记录；取消客户自提后恢复默认回单规则 |
| 后端审批联动 | 保持前后段外请进入审批、自运不审批 | 前段外请与后段外请继续沿用外请审批流，自运场景不额外进入审批 |
| 零担统一工作台 | 增加“客户自送到站确认”入口 | 在待询价/已询价相关操作菜单中增加前段客户自送入口，并在存在派车或前段外请时进行前端拦截提示 |
| 零担统一工作台 | 增加“客户自提”标记入口 | 在进行中阶段增加后段客户自提按钮，并同步在已完成阶段禁止继续进入回单押金跟踪 |
| 零担统一工作台 | 增加自动推断模式展示 | 在列表中补充前后段模式标签，并为客户自提场景展示“不生成回单”的提示 |

## 规则实现说明

### 1. 前段模式自动推断

| 前段模式 | 触发依据 |
| --- | --- |
| 自运 | 零担主单存在前段派车记录 |
| 外请 | 已创建前段外请子链 |
| 客户自送 | 已确认到站且不存在派车记录，同时带有客户自送到站确认标记 |

### 2. 后段模式自动推断

| 后段模式 | 触发依据 |
| --- | --- |
| 货站包送 | 默认主链继续跟踪后段 |
| 外请送货 | 已创建后段外请子链 |
| 客户自提 | 已标记客户自提 |

### 3. 回单规则联动

| 场景 | 回单处理 |
| --- | --- |
| 前段任意方式到站 | 仍保留前段到站回单规则 |
| 后段货站包送 | 正常生成/跟踪回单 |
| 后段外请送货 | 回单责任转移到后段外请责任单 |
| 后段客户自提 | 不生成回单；若已有主单回单记录则自动清理 |

### 4. 模式切换清理

| 操作 | 系统动作 |
| --- | --- |
| 标记客户自提 | 主单停止承担回单责任，并清理既有主单回单记录 |
| 取消客户自提 | 恢复后段默认回单规则 |
| 标记客户自送到站 | 自动打上前段客户自送标记，并补齐到站确认时间/人 |
| 取消客户自送到站 | 去除客户自送标记，交由实际操作重新推断 |
| 取消外请子链 | 主单模式自动回落到默认推断结果 |

## 备注

本次修改未引入新的人工模式下拉选择，而是严格遵循“由操作行为推断模式”的设计思路。对外请场景的审批流保持原有约束，对自运与客户自送/客户自提则通过标记与子链状态自动切换模式与回单规则。
