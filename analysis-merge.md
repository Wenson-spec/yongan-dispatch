# 询价发运台 vs 零担综合工作台 功能对比

## 询价发运台 (LtlInquiryStation) - 3个Tab
| Tab | 功能 |
|-----|------|
| 待询价 | 显示pending_inquiry订单，操作：创建询价（填货站/单价/送货费/其他费）、合并计划号分组 |
| 已询价 | 显示inquiry_confirmed订单，操作：上传开单(OCR识别+运费校准)、查看照片、备注/加急显示 |
| 零担派车 | LtlDispatch组件，操作：批量选单+填车辆司机→创建派车批次(自动推进dispatched)、派车记录、批量筛选 |

## 零担综合工作台 (LtlWorkspace) - 3个Tab
| Tab | 功能 |
|-----|------|
| 零担台账 | 全部零担订单汇总表，含费用统计卡片(总运费/总送货费/总费用)、详细列表、导出CSV/Excel |
| 进行中 | dispatched/shipped/in_transit/delivered状态，操作：上传货站运费图片(OCR)、编辑货站信息、查看单据、退回、删除、拖拽上传、备注/加急 |
| 已完成 | signed/settled状态，操作：退回、删除、备注/加急 |

## 重叠功能
1. 上传货站开单图片(OCR) - 两边都有
2. 编辑货站信息 - 两边都有
3. 查看已上传照片 - 两边都有
4. 备注/加急显示 - 两边都有
5. 状态标签显示 - 两边都有

## 合并方案
合并为统一的"零担工作台"，Tab结构：
1. **待询价** - 原询价发运台的待询价Tab
2. **已询价** - 原询价发运台的已询价Tab（含上传开单、运费校准、查看照片）
3. **零担派车** - 原询价发运台的零担派车Tab（LtlDispatch组件）
4. **进行中** - 原零担综合工作台的进行中Tab（已发运/运输中/已送达）
5. **已完成** - 原零担综合工作台的已完成Tab（已签收/已结算）
6. **零担台账** - 原零担综合工作台的零担台账Tab（全量汇总+导出）

这样一个页面就能覆盖零担订单的完整生命周期。
