## 三、发现的问题

### 🔴 问题1：PodDepositStation中残留大量未使用的押金代码

**严重程度：低（不影响功能，但浪费资源和影响代码可读性）**

押金Tab已迁移到找车台，但PodDepositStation中仍残留以下未使用的代码：
- `depositStatusFilter` 状态变量
- `refundDepositMut` mutation
- `depositSearchParams` 查询参数
- `depositData` / `depositLoading` / `refetchDeposit` 数据查询
- `depositStats` / `refetchStats` 统计查询
- `depositOrders` / `pendingRefund` / `filteredPendingRefund` 数据处理
- `refundedOrders` / `nonRefundable` / `filteredDoneDeposit` 数据处理
- `handleRefundDeposit` / `markNotRefundable` 操作函数
- `stats` 变量（基于depositStats）
- `refetchAll` 中包含 `refetchDeposit()` 和 `refetchStats()` 调用
- 押金导出CSV函数（行229-237）

这些代码虽然不影响编译和运行，但会产生**不必要的API请求**（每次打开回单管理台都会查询押金数据），浪费服务器资源。

### 🟡 问题2：连锁客服工作台台账导出功能未实现

**严重程度：中**

ChainWorkspace.tsx 第134行：`toast.info("台账导出功能即将上线")`

导出按钮存在但点击后只显示"即将上线"提示，功能未实际实现。

### 🟡 问题3：PlaceholderPage未被任何路由使用

**严重程度：低**

PlaceholderPage.tsx 存在但未被任何路由引用，属于死代码。

### 🟡 问题4：ComponentShowcase页面（1437行）

**严重程度：低**

这是一个UI组件展示页面，仅用于开发调试，不是业务功能。在生产环境中可以考虑移除。

## 四、关于"应收应付/追款雷达/异常监控/到付资金池"

经过全面排查，**这些模块从未在本项目中实现过**。具体证据：

1. **无前端页面文件**：没有任何以 Receivable/Payable/CollectionRadar/AnomalyMonitor/CodPool 命名的页面文件
2. **无后端路由**：server/routers/ 目录中没有相关路由文件
3. **无数据库表**：drizzle/schema.ts 中没有应收/应付/发票相关的表定义
4. **无路由注册**：App.tsx 中没有相关路由
5. **todo.md中无实现记录**：搜索整个todo.md，只有本次排查任务中提到这些模块，没有任何"已完成"的记录

**结论**：这些模块属于**尚未开发的功能需求**，不是"丢失"的功能。如果需要，可以作为新功能开发。
