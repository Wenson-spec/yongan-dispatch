# 永安调度系统第一批核心改动说明

## 本批修改文件

| 文件路径 | 改动主题 |
| --- | --- |
| `server/routers/pod.ts` | 修复回单状态更新越权、补充状态机校验、修正 `allPodsReceived` 聚合语义 |
| `server/routers/approval.ts` | 修复审批列表 `total` 与实际条数不一致 |
| `client/src/pages/OrderPool.tsx` | 收口订单池行级菜单、批量操作与退回入口 |
| `client/src/pages/FindVehicle.tsx` | 收口找车台页签与回单押金相关入口 |
| `client/src/pages/PodDepositStation.tsx` | 承接回单确认与退押金闭环能力，并将退回下沉到二级菜单 |

## 详细改动说明

### 1. 回单接口越权与状态机修复

在 `server/routers/pod.ts` 中，对回单状态更新能力进行了收口。核心处理包括：

| 改动点 | 说明 |
| --- | --- |
| 字段级鉴权 | 外请调度员仅允许执行“标记寄出”类更新，不再允许直接将回单改为“已收到” |
| 状态机约束 | 新增状态流转校验，不允许从“待寄”等中间状态直接跳到“已收到” |
| 批量确认限制 | 批量确认收到仅允许对满足前置状态的记录执行，避免跳步流转 |
| 幂等处理 | 对重复提交和无实际状态变化的更新做了保护，避免产生空更新和重复同步 |

### 2. 审批列表 `total` 统计修复

在 `server/routers/approval.ts` 中，将孤儿审批清理前置，并统一了列表数据与 `total` 统计口径。

> 修复后，审批关联订单已删除时，会先完成无效审批清理，再返回列表与总数，避免出现页面展示条数与 `total` 不一致的问题。

### 3. `allPodsReceived` 聚合语义修复

在 `server/routers/pod.ts` 中修正了回单聚合判断口径。

| 原问题 | 修复后语义 |
| --- | --- |
| “已寄出”也被计入“已全部收到” | 只有同组回单全部达到真实“已收到”状态时，`allPodsReceived` 才会返回 `true` |

### 4. 订单池 `OrderPool.tsx` 收口

在 `client/src/pages/OrderPool.tsx` 中，对默认界面暴露的深度流程能力进行了收口。

| 改动项 | 说明 |
| --- | --- |
| 行级菜单精简 | 去掉“标记送达、标记签收、改派、挂起、取消”等深流程操作 |
| 保留核心入口 | 保留“查看、编辑、跳转到所属工作台” |
| 批量操作收起 | 批量删除、批量退回、批量状态推进调整为仅管理员可见，并从默认界面收起 |
| 退回入口下沉 | 退回弹窗与强流程逆转入口下沉到二级菜单，降低误操作风险 |

### 5. 找车台 `FindVehicle.tsx` 收口

在 `client/src/pages/FindVehicle.tsx` 中，将页面重新聚焦为三个核心环节。

| 改动项 | 说明 |
| --- | --- |
| 页签收口 | 收敛为“待找车、待审批、确认派车”三个核心页签 |
| 功能迁移 | 回单处理、押金已处理、超期回单等功能迁移至回单押金台 |
| 只读展示 | 找车台仅保留回单/押金状态只读展示，不再直接执行退押金 |
| 超期监控收口 | 将超期回单改为顶部统计卡或跳转入口，避免在找车台继续展开闭环处理 |

### 6. 回单押金台 `PodDepositStation.tsx` 增强

在 `client/src/pages/PodDepositStation.tsx` 中承接从找车台迁移过来的闭环能力。

| 改动项 | 说明 |
| --- | --- |
| 页面职责升级 | 页面定位调整为“回单确认 + 退押金”统一闭环入口 |
| 退押金迁入 | 已收回单页签支持单笔、整组、批量退押金 |
| 保持回单确认 | 保留待收回单确认收到的原有能力 |
| 退回入口下沉 | 将单笔、分组、批量退回统一收敛到二级菜单 |
| 提示文案更新 | 页面说明同步调整，明确闭环入口已迁入当前工作台 |
